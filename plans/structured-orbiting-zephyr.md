# /services 플로우 다이어그램 테스트

## Context
서비스 페이지에 Spline 레퍼런스 스타일의 플로우 다이어그램을 적용하고 싶음.
레퍼런스: 아이소메트릭 그리드 위에 3D 칩 노드 4개가 네온 와이어로 순차 연결, 스크롤 시 와이어가 흘러가며 각 노드를 활성화.

구현: SVG 패스 + CSS 애니메이션 (stroke-dashoffset로 와이어 흐름 + 글로우 필터)

## 테스트 페이지
- `src/app/(public)/services-test/page.tsx` (새 파일)
- 3가지 배치 variant를 한 페이지에서 비교

```

│                                         │
│  OUR SERVICES 타이틀                    │
│                                         │
│  ┌──[01 교육]──wire──[02 솔루션]──┐     │
│  │                                │     │
│  └──[03 플랫폼]──wire──[04 MSP]──┘     │
│                                         │
│  각 노드: 글로우 색상 (blue→violet→     │
│           amber→emerald)                │

└─────────────────────────────────────────┘
→ 아래에 기존 상세 섹션 유지
```

## 플로우 다이어그램 공통 구현

### SVG 구조
- **그리드 배경**: CSS `repeating-linear-gradient` 45도 (아이소메트릭 느낌)
- **노드 4개**: 둥근 사각형 (rounded-xl), 각 서비스 색상 글로우
  - 비활성: bg-slate-200, 그림자 약함
  - 활성: 서비스 색상 배경 + box-shadow 글로우 + scale(1.05)
- **와이어 (SVG path)**: 노드 간 곡선 연결
  - `stroke-dasharray` + `stroke-dashoffset` 애니메이션으로 흐름 표현
  - `filter: drop-shadow(0 0 6px cyan)` 네온 글로우
  - 순차 delay (node1→wire1→node2→wire2→...)
- **라벨**: 각 노드 위/아래에 서비스명 + 서브타이틀

### 애니메이션
- Framer Motion `whileInView` 트리거
- 또는 Intersection Observer로 스크롤 위치 기반 순차 활성화
- 총 시퀀스: 약 3~4초 (노드당 ~0.8초)

### 반응형
- Desktop: 가로 4노드 배치
- Mobile: 세로 4노드 배치 (와이어도 세로로)
