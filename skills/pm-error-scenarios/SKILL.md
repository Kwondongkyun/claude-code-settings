# 에러/상태 시나리오

## 상태 정의 필수 원칙

모든 화면/기능에는 다음 5가지 상태를 반드시 정의한다:

| 상태 | 설명 | 필수 여부 |
|------|------|----------|
| **Default** | 정상 데이터가 있는 기본 상태 | 필수 |
| **Loading** | 데이터를 불러오는 중 | 필수 |
| **Empty** | 데이터가 없는 상태 | 필수 |
| **Error** | 에러가 발생한 상태 | 필수 |
| **Success** | 작업이 성공적으로 완료된 상태 | 해당 시 |

## 에러 유형별 정의

### 1. 유효성 검사 에러 (Validation Error)

사용자 입력이 규칙에 맞지 않을 때 발생한다.

**정의 항목:**
```markdown
| 필드 | 규칙 | 에러 메시지 | 검증 시점 |
|------|------|-----------|----------|
| 이메일 | 이메일 형식 | "올바른 이메일 형식으로 입력해주세요" | 포커스 아웃 |
| 비밀번호 | 8자 이상 + 특수문자 | "비밀번호는 8자 이상, 특수문자를 포함해야 합니다" | 실시간 |
| 이름 | 필수 입력 | "이름을 입력해주세요" | 제출 시 |
```

**검증 시점 가이드:**
- **실시간 (타이핑 중)**: 비밀번호 강도, 글자 수 제한
- **포커스 아웃 (필드 이탈)**: 이메일 형식, 중복 체크
- **제출 시**: 필수값 누락, 서버 검증 필요한 항목

### 2. 시스템 에러 (System Error)

서버/네트워크 문제로 발생한다.

| HTTP 상태 | 상황 | 사용자 메시지 | 복구 방법 |
|-----------|------|-------------|----------|
| 400 | 잘못된 요청 | "요청을 처리할 수 없습니다. 다시 시도해주세요." | 재시도 버튼 |
| 401 | 인증 만료 | "로그인이 만료되었습니다." | 로그인 페이지 이동 |
| 403 | 권한 없음 | "접근 권한이 없습니다." | 이전 페이지 또는 홈으로 이동 |
| 404 | 페이지 없음 | "요청하신 페이지를 찾을 수 없습니다." | 홈 이동 + 검색 제안 |
| 500 | 서버 에러 | "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요." | 재시도 버튼 + 고객센터 링크 |
| timeout | 응답 지연 | "응답 시간이 초과되었습니다." | 재시도 버튼 |
| offline | 네트워크 끊김 | "인터넷 연결을 확인해주세요." | 연결 상태 모니터링 + 자동 재시도 |

### 3. 권한 에러 (Permission Error)

사용자가 허용되지 않은 리소스에 접근할 때 발생한다.

**정의 항목:**
- 비로그인 사용자가 로그인 필요 페이지 접근 → 로그인 유도 후 원래 페이지로 복귀
- 권한 부족 사용자가 관리자 페이지 접근 → 권한 안내 + 홈으로 이동
- 삭제된 리소스 접근 → 안내 메시지 + 목록으로 이동

### 4. 비즈니스 에러 (Business Logic Error)

비즈니스 규칙 위반 시 발생한다.

**예시:**
- 재고 없는 상품 구매 시도
- 중복 신청
- 마감된 이벤트 참여
- 잔액 부족

## 에러 메시지 원칙

### 작성 규칙

1. **문제를 명확히 설명한다**
```
❌ "오류가 발생했습니다"
✅ "이미 등록된 이메일 주소입니다"
```

2. **해결 방법을 반드시 포함한다**
```
❌ "결제에 실패했습니다"
✅ "결제에 실패했습니다. 카드 정보를 확인하거나 다른 결제 수단을 이용해주세요."
```

3. **사용자 탓하지 않는다**
```
❌ "잘못된 비밀번호입니다"
✅ "비밀번호가 일치하지 않습니다. 다시 입력해주세요."
```

4. **기술 용어를 사용하지 않는다**
```
❌ "ETIMEDOUT: connection timed out"
✅ "서버 응답이 지연되고 있습니다. 잠시 후 다시 시도해주세요."
```

### 배치 규칙

| 에러 유형 | 표시 위치 | UI 컴포넌트 |
|-----------|----------|------------|
| 필드 에러 | 해당 필드 바로 아래 | 인라인 텍스트 (빨간색) |
| 폼 에러 (여러 필드) | 폼 상단 | Alert/Banner |
| API 에러 | 화면 상단 또는 중앙 | Toast 또는 Alert |
| 치명적 에러 | 전체 화면 | 에러 페이지 (404, 500) |
| 확인 필요 에러 | 모달 | Dialog |

### 심각도별 처리

| 심각도 | 예시 | UI | 사용자 행동 |
|--------|------|-----|-----------|
| 낮음 | 입력 형식 오류 | 인라인 메시지 | 즉시 수정 가능 |
| 중간 | API 실패 | Toast (자동 사라짐) | 재시도 또는 무시 |
| 높음 | 세션 만료 | Modal (확인 필요) | 반드시 대응 필요 |
| 치명적 | 서버 다운 | 전체 에러 페이지 | 대기 또는 이탈 |

## 로딩 상태

### 로딩 유형

| 유형 | 사용 시점 | UI |
|------|----------|-----|
| **Determinate** (확정적) | 진행률 알 수 있을 때 (파일 업로드) | Progress Bar |
| **Indeterminate** (불확정) | 소요 시간 불확정 (API 호출) | Spinner |
| **Skeleton** | 콘텐츠 로딩 시 | Skeleton UI (콘텐츠 형태의 회색 블록) |
| **Optimistic** | 성공 확률 높은 작업 | 즉시 반영 후 백그라운드 처리 |

### 로딩 시간별 처리

| 소요 시간 | 처리 |
|-----------|------|
| 0-300ms | 아무것도 표시하지 않음 (깜빡임 방지) |
| 300ms-1s | Spinner 또는 Skeleton 표시 |
| 1s-10s | Progress 표시 + 안내 텍스트 |
| 10s+ | 취소 옵션 제공 + 백그라운드 처리 안내 |

## 빈 상태 (Empty State)

빈 상태는 단순히 "데이터 없음"이 아니라 사용자를 다음 행동으로 유도해야 한다.

### 빈 상태 유형

| 유형 | 상황 | 필수 요소 |
|------|------|----------|
| **첫 사용** | 아직 데이터를 생성하지 않음 | 설명 + CTA ("첫 번째 항목 만들기") |
| **검색 결과 없음** | 검색/필터 결과 0건 | 안내 + 대안 제시 ("검색어를 변경해보세요") |
| **삭제 후 빈 상태** | 모든 항목 삭제 후 | 설명 + 새로 만들기 CTA |
| **권한 없음** | 볼 수 있는 데이터 없음 | 권한 안내 + 요청 방법 |

### 빈 상태 작성 원칙

```
❌ Bad: "데이터가 없습니다"
✅ Good: "아직 등록된 프로젝트가 없습니다. 첫 번째 프로젝트를 만들어보세요."
         [+ 새 프로젝트 만들기] 버튼
```

- 왜 비어있는지 설명
- 다음에 할 수 있는 행동 제시 (CTA)
- 가능하면 일러스트/아이콘으로 시각적 안내

## 엣지케이스별 대응

### 오프라인 상태
- 현재 오프라인임을 상단 배너로 알림
- 이전에 불러온 데이터는 계속 표시 (캐시)
- 쓰기 작업은 큐에 저장 후 재연결 시 동기화
- 재연결 시 자동으로 배너 제거

### 긴 텍스트/데이터
- 텍스트: 말줄임(...) + "더보기" 버튼
- 이름/제목: 최대 표시 길이 정의 (예: 최대 50자)
- 목록: 페이지네이션 또는 무한 스크롤
- 테이블: 가로 스크롤 또는 반응형 축소

### 동시 편집
- 다른 사용자가 편집 중임을 알림
- 저장 시 충돌 감지 → "최신 버전으로 업데이트" 또는 "내 변경사항 유지" 선택
- 자동 저장 시 최후 저장자 우선 (Last Write Wins) 또는 병합

### 브라우저 뒤로가기
- 폼 작성 중 뒤로가기 → 확인 대화상자 ("변경사항이 저장되지 않습니다")
- 멀티스텝 프로세스 → 이전 단계로 이동 (데이터 유지)
- 모달/드로어 열림 상태 → 모달만 닫기 (페이지 이동 아님)

### 세션 만료
- 토큰 만료 시 → 자동 갱신 시도 (Refresh Token)
- 갱신 실패 시 → "세션이 만료되었습니다" 모달 + 로그인 유도
- 로그인 후 원래 작업 중이던 페이지로 복귀

## 상태 정의 템플릿

```markdown
## [기능명] 상태 정의

### Default (기본)
- 화면 구성: ...
- 표시 데이터: ...

### Loading (로딩)
- 로딩 UI: Skeleton / Spinner / Progress
- 로딩 중 사용자 행동 제한: 있음/없음

### Empty (빈 상태)
- 메시지: "..."
- CTA: [버튼 텍스트]
- 이미지/아이콘: 있음/없음

### Error (에러)
- 에러 유형: 네트워크 / 서버 / 권한
- 에러 메시지: "..."
- 복구 방법: 재시도 / 페이지 이동 / 고객센터

### Success (성공)
- 피드백: Toast / 페이지 이동 / 인라인 메시지
- 다음 동작: ...
```
