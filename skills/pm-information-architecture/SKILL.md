# 정보 구조 설계

## IA 4대 구성 요소

정보 구조(Information Architecture)는 4가지 체계로 구성된다.

### 1. 조직 체계 (Organization Systems)

콘텐츠를 어떤 기준으로 분류할지 결정한다.

| 분류 방식 | 설명 | 적합한 경우 |
|-----------|------|------------|
| **주제별** | 콘텐츠의 주제/카테고리 | 쇼핑몰 카테고리, 문서 분류 |
| **작업별** | 사용자가 수행하는 작업 | 관리자 페이지 (사용자 관리, 설정, 통계) |
| **대상별** | 사용자 유형/역할 | 학생/교사/관리자 별 메뉴 |
| **시간별** | 시간순 정렬 | 게시판, 활동 로그, 타임라인 |

**선택 기준:**
- 사용자가 "무엇을 찾는가" → 주제별
- 사용자가 "무엇을 하려 하는가" → 작업별
- 사용자 유형이 명확히 다른가 → 대상별

### 2. 라벨링 체계 (Labeling Systems)

메뉴, 버튼, 카테고리 등의 이름을 어떻게 지을지 결정한다.

**규칙:**
- 사용자 친숙한 용어 사용 (기술 용어 금지)
- 같은 계층의 라벨은 동일한 문법 구조 (명사형이면 모두 명사형)
- 짧고 명확하게 (2-4단어 이내)
- 모호한 라벨 금지 ("기타", "관리", "정보")

```
❌ Bad
- 시스템 관리
- 기타 설정
- 정보

✅ Good
- 사용자 관리
- 알림 설정
- 공지사항
```

**라벨링 체크리스트:**
- [ ] 사용자가 클릭 전에 무엇이 나올지 예측 가능한가?
- [ ] 같은 계층의 라벨이 문법적으로 일관적인가?
- [ ] 약어 없이 이해 가능한가?
- [ ] 다른 라벨과 의미가 겹치지 않는가?

### 3. 내비게이션 체계 (Navigation Systems)

사용자가 콘텐츠 사이를 이동하는 방식을 설계한다.

**내비게이션 유형:**

| 유형 | 위치 | 용도 |
|------|------|------|
| **글로벌 네비게이션** | 상단/사이드바 | 전체 서비스의 주요 섹션 이동 |
| **로컬 네비게이션** | 섹션 내 | 현재 섹션 내 하위 페이지 이동 |
| **컨텍스트 네비게이션** | 콘텐츠 내 | 관련 콘텐츠 링크 (추천, 관련 글) |
| **유틸리티 네비게이션** | 우측 상단 | 보조 기능 (프로필, 설정, 로그아웃) |
| **브레드크럼** | 상단 | 현재 위치와 상위 경로 표시 |
| **페이지네이션** | 하단 | 목록 페이지 간 이동 |

**네비게이션 설계 규칙:**
- 글로벌 네비게이션 항목은 5-7개 이내
- 최대 3단계 깊이 (3클릭 규칙)
- 현재 위치 항상 표시 (활성 메뉴 하이라이트)
- 모바일에서는 햄버거 메뉴 또는 하단 탭으로 축소

### 4. 검색 체계 (Search Systems)

콘텐츠가 많을 때 사용자가 원하는 정보를 찾는 방식을 설계한다.

**검색 설계 가이드:**
- 데이터 100건 이상이면 검색 기능 필수
- 검색 결과에 필터/정렬 제공
- 자동완성/추천 검색어 지원
- "검색 결과 없음" 상태에 대안 제시
- 최근 검색어 기록 제공

## 사용자 멘탈 모델

사용자가 정보를 어디에서 찾을 것으로 기대하는지 고려한다.

### 멘탈 모델 적용 규칙

1. **예측 가능한 위치에 배치**
   - 로그아웃 → 우측 상단
   - 검색 → 상단 중앙 또는 우측
   - 메인 액션(CTA) → 우측 또는 하단
   - 취소 → 확인 버튼 좌측

2. **기존 서비스와 유사한 패턴 사용** (Jakob's Law)
   - 장바구니 아이콘 → 우측 상단
   - 프로필/설정 → 우측 상단
   - 뒤로가기 → 좌측 상단

3. **정보 가까이 배치**
   - 에러 메시지 → 해당 필드 바로 아래
   - 도움말 → 관련 기능 옆
   - 수량 변경 → 상품 옆

## 3클릭 규칙

> 사용자가 원하는 정보에 3번 이내의 클릭으로 도달할 수 있어야 한다.

**적용 방법:**
- 핵심 기능은 1-2클릭 이내 접근 가능
- 3단계 이상의 메뉴 depth 피하기
- 자주 사용하는 기능에 단축 경로 제공 (대시보드, 즐겨찾기)
- 브레드크럼으로 상위 경로 바로 이동 가능

**예외:** 3클릭을 초과해도 되는 경우는 "실수 방지"가 클릭 수보다 중요할 때.
- 결제/설정 변경: 확인 페이지를 추가해서 실수로 결제되는 것을 방지
- 회원가입/주문: 9단계 이상이므로 단계를 나누되 진행률 표시 (몇 단계 남았는지)

## 사이트맵 구조

새 프로젝트 또는 주요 리뉴얼 시 사이트맵을 먼저 작성한다.

### 사이트맵 템플릿

```markdown
## [서비스명] 사이트맵

### 메인 (/)
├── 홈 (/)
├── 로그인 (/login)
└── 회원가입 (/register)

### 대시보드 (/dashboard)
├── 개요 (/dashboard)
├── 통계 (/dashboard/analytics)
└── 알림 (/dashboard/notifications)

### 설정 (/settings)
├── 프로필 (/settings/profile)
├── 계정 (/settings/account)
└── 알림 설정 (/settings/notifications)
```

### 사이트맵 작성 체크리스트

- [ ] 모든 페이지가 1개 이상의 경로에서 접근 가능한가?
- [ ] 고아 페이지(orphan page)가 없는가?
- [ ] URL 구조가 계층을 반영하는가?
- [ ] 핵심 페이지는 2클릭 이내에 접근 가능한가?
- [ ] 모바일에서도 동일한 구조가 유효한가?

## 메뉴/카테고리 네이밍 원칙

1. **구체적으로** — "관리" → "사용자 관리", "주문 관리"
2. **동사보다 명사** — "관리하기" → "사용자 관리"
3. **일관된 형식** — 같은 레벨은 같은 문법 구조
4. **겹치지 않게** — 카테고리 간 경계가 명확해야 함
5. **사용자 용어** — 내부 용어가 아닌 사용자가 쓰는 단어
